🚀 Deploy & Troubleshoot

Step-by-Step Setup

1 Go to your repo on GitHub
2 Click Settings tab (⚙️ at top)
3 Click Pages in left sidebar
4 Under "Build and deployment":
• Source: Deploy from a branch
• Branch: main / (root)
• Click 💚 SAVE
5 Wait 1-2 minutes ⏳
✓ Your URL will appear at the top!

Expected URL Format

https://USERNAME.github.io/REPO-NAME/

Example:

If username is "john" and repo is "my-app":

https://john.github.io/my-app/

The URL is case-sensitive! Use exact repo name.

☐ URL Issues

Fix 1: Re-save

Go to Settings → Pages and click Save again. Sometimes it needs a refresh.

Fix 2: New Commit

Make any small change to index.html and commit. This triggers redeployment.

Fix 3: Check Location

Ensure index.html is at the ROOT of your repo, not in a docs/ folder.

🔥 Most Common Fix

Your repository is likely PRIVATE. GitHub Pages only works on public repositories with free accounts.

To fix: Settings → General → Danger Zone → Change visibility → Make public

Or upgrade to GitHub Pro ($4/month) for private Pages

How to Use Your AI Agent

🎨 Creating Images

Just type what you want to create. Be descriptive! Example: "A serene Japanese garden with cherry blossoms at sunset, watercolor style"

💬 Chat Commands

The agent understands natural language. Try: "make it brighter", "change to anime style", "create a variation"

📎 Upload & Edit

Upload images using the paperclip button. Ask the agent to modify them: "add a sunset background" or "make it look like a painting"

After generating, use quick actions: Enhance, Upscale, Create Variations, or Change Style
